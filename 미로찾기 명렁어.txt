다음 조건으로 스도쿠 게임 구현 계획을 plan.md에 정리해줘, 단 구현을 절때 해서는 안돼

기술 스택: React 최신 버전, TypeScript 5, Vite 5, Tailwind CSS
상태 관리: useReducer + Context API (외부 상태 라이브러리 없음)
컴포넌트: Atomic Design (atoms/molecules/organisms)
경로 별칭: @/ (tsconfig paths와 vite resolve.alias 동기화)

계획에 포함할 것:
- 디렉토리 구조
- 타입 정의 (types/sudoku.ts)
- Reducer 액션 목록
- 컴포넌트별 props 인터페이스

[기능 요구사항]
1. 처음에 스도쿠처럼 난이도를 선택하는 화면을 띄운다. (난이도에 따라 쉬움부터 어려움까지 미로의 크기가 점차 커진다.)
2. 미로가 생성되면 사용자가 미로를 분석할 수 있는 시간이 주어지며 시간은 1분, 이때 사용자는 펜처럼 마우스로 화면을 그릴 수 있어 미로의 길을 찾을 수 있다.
3. 미로에는 사용자의 위치를 나타내는 원과 도착지를 나타내는 초록 모양의 칸이 주어진다.
3. 시간이 다 될 경우 원은 자동으로 앞으로 이동하게되고 사용자는, 방향키로 방향을 조작해 원을 조작한다.
4. 이때 만약 사용자가 벽에 부딪힐 경우 혹은 왔던 길로 되돌아갈 경우 게임 오버가 된다. (게임 오버가 되면 팝업을 띄우고 새 게임 버튼이 나오게 되고, 새 게임 버튼을 누르면 다시 난이도 선택 화면으로 이동하게 된다.)
5.사용자가 원을 조작하여 도착지(초록 칸)에 도달하게 되면 게임 클리어가 된다. (게임 오버가 되면 팝업을 띄우고 새 게임 버튼이 나오게 되고, 새 게임 버튼을 누르면 다시 난이도 선택 화면으로 이동하게 된다.)
